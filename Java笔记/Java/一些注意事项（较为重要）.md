## 而一般工具类中的方法才定义为static，static代表属于这个类而非对象
- 定义在属性上，可以被所有对象共享
- 定义在方法上，可以通过类名直接调用
- 定义在类上，成为静态内部类

## 程序设计要求
- 高内聚
	- 一个模块或类专注于一个具体的任务和功能，相关的操作都封装在同一个模块中，让该模块更独立和更容易理解
- 低耦合
	- 不同模块间的依赖关系尽量少，让一个模块的变动尽量不影响其他模块，增加代码的灵活性和维护性


## 访问修饰符：
- （注意）类的修饰符只有default 和 public
- public(任何地方都可访问)
	- protect（只能在  子类>同一包中>同一个类    中访问）
		- default（没有加任何东西就默认是这个，只能在  同一包中>同一个类     中访问）
			- private （只能在   同一个类      中访问）

## final:
- 最终的意思
- 修饰  变量  时指该值是常量，不可变
- 修饰  方法  时，该方法不能重写
- 修饰  类  时，表示该类不能被继承

## 有时查看源码时，会发现native关键字，这是本地的意思，关于硬件设备的，不必过多关注

## 方法重载
- 不止构造器可以重载，在一个类中其他方法也可以
- 规则都是以形参的变化有关：
	- 1、个数不同
	- 2、类型不同
	- 3、顺序不同


## 方法重写（Override）
- 必须要有[[继承]]关系，当父类 的方法不能满足子类需求的时候可以重写该方法（覆盖）
- 重写后会优先调用子类
- 子类重写的方法不允许比父类方法更小的权限（两者都没有private）
